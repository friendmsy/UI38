1、采用PO模式，页面对象化
2、框架
bin		是存放项目执行文件
case	是存放用例文件
common	是存放通用文件
file	可存在上传文件
log		存放日志类及生成的日志文件
page    存入页面对象
report	存放输出的测试执行
config	定义基本属性，主要是定义上述模块的路径，如果要修改直接修改定义的路径即可
yamlconfig.yml 配置文件

3、可excel参数化自由组合场景，关键字是方法，根据传入的方法调用对应的页面对象方法，
所以每个方法最好设置成自动退出浏览器，因为组合场景可能会用到不同的页面对象，对象
不同，实例不同，所以上一个实例结束最好要关闭浏览器

4、自动化目的是看功能是否有问题，所以可以尽量少参数化，预留参数化的窗口更多是方便平时辅助测试

5、使用pytest多进行且要生成测试报告的情况下，要在report目录下执行命令，这样生成的报告才能发送到邮件

6、case 目录下除了用例文件，不要再放其他文件夹，因为代码中设计是在运行用例前会把case目录下的文件夹先删除，不删除case目录下的文件

7、report 目录下的报告在每次运行用例前会删除旧的报告，方式是删除整个目录下的所有文件及文件夹，以保证此次发送到邮箱的报告是此次运行的报告
